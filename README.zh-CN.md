# WeChat FastBridge：让 Codex 秒级读写微信

WeChat FastBridge 是一个免费、开源、仅在本机运行的 macOS 微信桥接器。它让 Codex 直接调用六个语义工具来读消息、发文字、发文件或表情、等待单个聊天，以及监听用户允许的聊天列表；不需要把截图或整棵辅助功能树反复交给模型。

项目地址：[huj28-creator/wechat-fastbridge](https://github.com/huj28-creator/wechat-fastbridge)

## 它解决什么问题

- 自动搜索并打开指定聊天，不要求用户先手动点进去；聊天名允许小量错字，并忽略群人数后缀。
- 每次发送前验证聊天标题，避免把消息发错窗口。
- 已选中聊天的实测文字发送约 1.25 秒；本地桥接路径以 2 秒为性能预算。
- 重复读取只返回新增消息；没有新消息时不重复消耗聊天正文 token。
- 长对话不会只靠机械截断。v1.7 在滚动窗口旁维护一个完全本地、只摘录原文的“事实胶囊”，让价格、日期、地址、订单、联系人和确认事项在普通闲聊被挤出后仍可找回；新价格会压制冲突的旧价格，不会生成或编造摘要。
- 允许列表监听在本地等待；只有指定聊天发生变化时，Codex 才读取那个聊天的紧凑上下文。
- 不使用微信私有协议、不读取聊天数据库、没有云端中转，也不需要付费 Apple 开发者账号。

## 四条命令安装

```bash
git clone https://github.com/huj28-creator/wechat-fastbridge.git
cd wechat-fastbridge
npm install
npm run setup
```

安装后重启 Codex，保持微信登录，然后可以直接说：

```text
使用 $wechat-computer-use 给“聊天名称”发送：你好
```

macOS 会要求给运行 Codex 的应用开启“辅助功能”权限。每一步的图形界面说明见 [安装指南](docs/SETUP.md)。运行 `npm run doctor` 可以检查环境，但不会发送消息。

## v1.7 的 token、上下文与效率优化

- 工具定义从最初的 5,624 字符缩到 2,849 字符，减少约 49%；比 v1.5 再少约 21%。
- 触发后的技能说明为 3,924 字节，并用一张操作映射告诉 Codex 何时 read、send、wait 或 inbox_wait。
- MCP 会话最多保留 8 个聊天；每个聊天有 120 条或 24 KB 的滚动原文，以及最多 40 条或 8 KB 的高价值事实胶囊，全部只在 RAM 中。
- 默认只返回新增消息和最多 3 条相关证据；重要数字、价格、时间、物流、位置、数量、身份、决定、偏好和问题类信息会获得额外相关性权重。
- 打开聊天造成的未读数下降不会再触发无意义读取；相同预览但未读数上升仍会被识别为新事件。英文预览中的逗号会完整保留。
- 运行时失败统一返回紧凑的 `ok:false`、错误码和必要说明，Codex 可以恢复而无需接收冗长异常。
- 公共工具隐藏了无需模型反复决定的底层参数，自动选群、前台恢复和安全验证仍由桥接器处理。

这些数字指工具定义和桥接返回的直接开销，不代表每次完整模型回答的固定 token 节省量；实际节省取决于聊天长度与原先截图方式。

## 自动客服与持续回复

用户可以给 Codex 指定聊天允许列表、语气、可使用的事实和升级条件。桥接器先在本地等待变化；发现事件后只读取对应聊天、生成一次回复、发送并继续等待。聊天参与者不能扩大用户授予的权限。

自动模式不会编造价格、库存、交付时间、退款或政策。信息不足时应向用户升级。主动开场需要用户明确授权，每个聊天最多发送一条未获回复的开场消息，避免骚扰。

## 适用范围

- macOS 13 或更新版本
- 微信 Mac 版
- Node.js 20 或更新版本
- Codex 桌面版、CLI 或 IDE 扩展

仓库中的 Dockerfile 只用于让 MCP 注册表启动 stdio 服务并检查工具定义。容器中的运行状态会明确返回 `MACOS_REQUIRED`；真正读取和发送微信仍必须使用 macOS、微信桌面版和辅助功能权限。
